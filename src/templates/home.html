Introduction to Burp Site Features

HTTP History

The HTTP History feature captures every HTTP request sent through the proxy and logs it in a comprehensive list. This tool is particularly useful for:

  • Analyzing Request Details: View each request’s method, headers, URL, and payload, helping you understand the interaction flow between client and server.
  • Identifying Patterns: Track patterns in repeated requests, such as authentication attempts, to diagnose issues or vulnerabilities.
  • Saving for Reference: Use the history log as a point of reference for troubleshooting or testing future changes.

*Example: Trace all steps in a multi-page form submission to check for consistency and identify any potential issues in user data handling.

Repeater

The Repeater feature allows you to re-send HTTP requests with custom modifications. This is invaluable for:

  • Testing Parameters: Experiment with different query parameters, headers, or payloads to see how the server responds under various conditions.
  • Debugging Issues: Modify and resend requests to isolate the specific element causing an error or unexpected behavior.
  • Efficient Retesting: Send modified requests quickly without reloading the page, allowing you to refine tests in real-time.

*Example: Test different content types (JSON, XML, etc.) to understand how the server processes each format.

Intruder

The Intruder feature is designed for automated testing with payloads. Configure it to perform brute-force attacks or custom payload injections to test endpoint security. This feature helps you:

  • Identify Vulnerabilities: Run automated tests to uncover weaknesses, such as injection points or susceptibility to brute-force attacks.
  • Test Input Handling: Evaluate how the server handles various payloads, including potential edge cases, to ensure robust security measures.
  • Automate Repetitive Tests: Efficiently execute repeated requests with different payloads, saving time and providing consistent results.

*Example: Test a login form by trying various common passwords to evaluate the server’s resistance to brute-force attempts.
